# mITT vs Allocated Audit — triage notes

Generated by `scripts/audit_mitt_vs_allocated.py`. CSV: `outputs/mitt_audit.csv`.

The audit walks every `*_REVIEW.html` dashboard, extracts each trial entry's
`tE/tN/cE/cN` quartet, and tags rows with heuristic flags. Flags are
**prompts for human review**, not verdicts.

## Headline counts

- 170 dashboards scanned
- 515 binary trials extracted
- 241 with at least one flag (47%)
- 183 BIG_TRIAL (n>=1000)
- 64 EQUAL_ARMS (tN==cN, both >=50)
- 11 ROUND_HUNDRED (tN or cN ends in 00)
- 8 MITT_MENTIONED (group/snippet text already explicit)

### 2) Cluster-RCT denominators — DIFFERENT issue (not mITT vs allocated)

`AZITHROMYCIN_CHILD_MORTALITY_REVIEW.html` flags MORDOR-I (95000/95000),
MORDOR-II (50000/50000), AVENIR (45000/45000). These are
**cluster-randomized** trials. The 95k figure is roughly the
person-years-at-risk in MORDOR-I (Keenan 2018 NEJM reports
89,891 / 89,587). The dashboard's binary engine would treat them as
individual-patient denominators, ignoring design effect — that is a
**different bug** than mITT-vs-allocated and should be tracked
separately. Mark as `CLUSTER_RCT_NEEDS_DEFF_ADJUSTMENT`. Not fixed in
this pass.

### 4) PEGCETACOPLAN_GA — addressed in this commit

OAKS (tN=202, cN=213) and DERBY (tN=206, cN=208) use **modified-ITT** denominators
matching the published primary (Heier 2023 Ophthalmology + Liao 2023 Lancet),
NOT randomized-allocation counts. The number 213 also appears in OAKS narrative
as the peg-Q2M *allocated* count, which is the source of the user's confusion.
Clarification notes added in PICO `out` field and OAKS/DERBY `group` fields.
