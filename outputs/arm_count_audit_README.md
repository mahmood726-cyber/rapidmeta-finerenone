# Portfolio arm-count audit — findings summary

Generated by `scripts/audit_review_arm_counts.py` after the pegcetacoplan
NCT-swap discovery (commit 3999c1fe). Scans every `*_REVIEW.html` for
inconsistencies between the trial-row `tN+cN` (engine input) and the
sum of `n=N` mentions in the Enrollment evidence narrative.

## Result: zero typos found

After filtering false positives (multi-arm trials where the row is a
2-arm contrast and the narrative lists all arms), the audit surfaced
3 findings — all of which are documented per-protocol or subgroup
designs, not typos:

| File | Trial | row vs narrative | Why it's fine |
|---|---|---|---|
| BELIMUMAB_SLE_REVIEW.html | BLISS-52 | 418 vs 1070 | Row uses 10 mg/kg vs placebo PP-set (n=205 vs 213); narrative lists all 3 ITT arms |
| COVID_ORAL_ANTIVIRALS_REVIEW.html | EPIC-HR | 1379 vs 3625 | Row uses mITT1 (treated within 3 days, n=1379); narrative lists full mITT (n=2246) AND mITT1 |
| IO_CHEMO_NSCLC_1L_REVIEW.html | IMpower150 | 692 vs 800 | Row uses EGFR/ALK WT subgroup (356/336); narrative lists both ITT (400) and WT subgroup |

In each case the narrative explicitly documents the row's denominator choice ("primary comparison for this app: …").

## Methodology limitations

This audit is purely structural — it does not call CT.gov to verify
NCT-vs-enrollment correctness. The pegcetacoplan NCT-swap (where
NCT03525613=OAKS and NCT03525600=DERBY were inverted in `nctAcronyms`)
was caught only by checking the published abstract via PubMed. A
follow-up CT.gov-cross-check sweep across all NCTs in the portfolio
would be the next-level audit; not done in this pass.
